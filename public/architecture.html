Tanpa sensor pintu, tetap realtime

Sistem laporan walk-in harian berbasis form, API, dan dashboard live.

Workflow Operasional

1

Master toko disiapkan

File Excel All Brand diekstrak menjadi master store yang dipakai form dan dashboard.

2

Store Leader submit harian

SL mengetik store code, nama toko terdeteksi, nama SL terisi otomatis, lalu mengisi jumlah customer per brand.

3

Data tersimpan dan bisa diperbarui

Jika toko submit ulang pada tanggal yang sama, sistem mengganti laporan lama agar tidak dobel hitung.

4

PIC monitor live

Dashboard memperlihatkan total walk-in, report rate, brand tertinggi, history 7 hari, dan toko belum report.

Kontrol Data

A

Validasi store code

Hanya store code di master yang diterima oleh form store.

B

Audit timestamp

Setiap laporan menyimpan tanggal report dan waktu submit untuk jejak update.

C

Single source of truth

Untuk preview, data tersimpan di JSON lokal. Versi produksi bisa memakai PostgreSQL, Supabase, Firebase, atau Google Sheets API.

D

Export lanjutan

Data harian dapat ditambah fitur export Excel/CSV, ranking toko, dan laporan bulanan.

Arsitektur Web Preview

Layer 1

Store Master

Excel April 2026 diekstrak ke `data/stores.json` untuk lookup dan daftar toko.

Layer 2

Frontend Store

`/form` dipakai Store Leader untuk input customer per brand.

Layer 3

Express API

`/api/submit`, `/api/data`, `/api/today`, `/api/stores`, dan `/api/stream`.

Layer 4

Data Store

`data.json` menyimpan transaksi harian selama preview localhost.

Layer 5

Dashboard PIC

`/dashboard` memvisualkan KPI, chart, feed realtime, dan pending toko.
